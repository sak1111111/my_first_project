1. Семантическая верстка 
Использует теги по смыслу (`<article>`, `<nav>`), а не только `<div>`. Важна для SEO и доступности.

2. `<article>` vs `<section>`  
`<article>` — самостоятельный контент (пост, новость). `<section>` — часть документа (глава, блок).

3. `<header>` и `<footer>`  
Для шапки и подвала. Да, можно использовать несколько раз (например, в каждой статье).

4. Основной контент
Тег `<main>`. `<div>` не передает смысл, только блок.

5. `<nav>` vs список ссылок**  
`<nav>` — для основной навигации (меню). Обычный список — просто ссылки. `<nav>` обязателен для главного меню.

6. Атрибут `datetime` в `<time>` 
Нужен для машинного чтения даты. Пример: `<time datetime="2023-10-01">1 октября</time>`.

7. `<figure>` и `<figcaption>`**  
Группируют изображение с подписью. Улучшают доступность и структуру.

8. `<aside>` 
Для бокового/вспомогательного контента (сайдбар, цитаты, реклама).

9. Влияние на SEO и доступность 
Поисковики лучше понимают контент. Скринридеры легнее ориентируются.

10. Использованные теги 
`<header>`, `<main>`, `<nav>`, `<article>`, `<section>`, `<footer>`. Помогают структурировать страницу для всех пользователей.
