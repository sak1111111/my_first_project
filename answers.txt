1. Три основные области работы с файлами в Git:

Рабочая папка - где ты работаешь с файлами
Индекс (staging) - подготовленные к коммиту файлы
Репозиторий - история всех коммитов
2. Что делают команды:

git add - добавляет файлы в индекс
git commit - сохраняет изменения из индекса в репозиторий
git push - отправляет коммиты на удалённый репозиторий
3. Информативные сообщения коммитов помогают понять, что было изменено и зачем, без просмотра кода.

4. git add имя_файла vs git add .:

git add имя_файла - добавляет конкретный файл
git add . - добавляет все изменённые файлы
Используй первый для выборочного добавления, второй - когда уверен во всех изменениях
5. git status показывает:

Изменённые файлы
Файлы в индексе
Неотслеживаемые файлы
6. git remote add origin URL связывает локальный репозиторий с удалённым. origin - стандартное имя для удалённого репозитория.

7. Флаг -u в git push -u origin main запоминает связь между ветками, чтобы потом использовать просто git push.

8. Ветка по умолчанию - main (раньше была master). Важно для первого push и при создании репозитория.

9. git log показывает историю коммитов: хеш, автора, дату и сообщение.

10. Если сделать git commit без git add - изменения не попадут в коммит, только уже добавленные в индекс файлы.
